# ClawShow MCP 平台战略文档

> 版本: 1.0 | 日期: 2026-04-02 | 状态: 战略规划阶段

---

## 一、核心定位

**"Instant Backend"** — 为没有后端的中小企业，通过AI提供平台级能力。

- **AI是客户端** — 用户通过AI对话触发操作
- **ClawShow是执行引擎** — MCP Tool执行具体任务，产生真实结果
- **FocusingPro是数据后端** — 十年积累的平台提供数据持久化（对AI不可见）

---

## 二、设计原则

| 原则 | 说明 |
|------|------|
| **零注册** | 客户不需要注册账号，第一次调用自动创建namespace |
| **零学习** | 没有管理后台，所有操作通过AI对话完成 |
| **有交付物** | 每次调用产生真实结果：URL、PDF、邮件、支付链接 |
| **数据有归属** | 持久化数据存于FocusingPro，客户可随时查询/导出/删除 |
| **Zero Human Intervention** | 调用完即完成，不需要人工步骤 |

### Tool准入标准

每个Tool必须同时满足以下四条：

| # | 标准 | 来源 | 说明 |
|---|------|------|------|
| 1 | **AI自己做不到** | ClawShow原标准 | 必须产生外部世界的真实变化 |
| 2 | **有可交付物** | ClawShow原标准 | 返回URL、PDF、确认等实际结果 |
| 3 | **零人工干预** | ClawShow原标准 | 调用完即完成 |
| 4 | **卖结果不卖工具** | Sequoia论点 | 客户付的是结果（"学费已收到"），不是工具（"收费软件"） |

## 三、三层架构

```
┌─────────────────────────────────────────────────────┐
│  第一层：ClawShow MCP（对AI可见）                      │
│  通用引擎 + 工作流，标准输入输出，无需认证               │
│  全球AI可调用（AI调AI + 人通过AI调用）                  │
├─────────────────────────────────────────────────────┤
│  第二层：FocusingPro 数据层（对AI不可见）               │
│  可选的数据持久化，通过 save_to_backend 参数启用        │
│  提供namespace数据隔离                                │
├─────────────────────────────────────────────────────┤
│  第三层：FocusingPro 完整功能（增值服务）               │
│  面试管理、排课、成绩系统等深度功能                     │
│  面向学校等深度客户，通过MCP或传统界面访问              │
└─────────────────────────────────────────────────────┘
```

### AI友好设计三原则

1. **Tool描述即广告** — 英文描述，让全球AI能发现和理解
2. **标准数据格式** — 输入输出用JSON/CSV/URL，不绑定特定系统
3. **无状态优先，有状态可选** — 默认纯执行，可选持久化到FocusingPro

## 五、工作流引擎（编排层）

### 工作流引擎 `run_workflow`

工作流不是和引擎并列的第七个引擎，而是**在引擎之上的编排层**。通过 `type` 参数切换不同业务流程，内部自动调用多个引擎协同完成任务。

```
┌──────────────────────────────────┐
│  ⑦ 工作流引擎 run_workflow        │  ← 编排层
│  type = payment_collection       │
│       | enrollment               │
│       | order_fulfillment        │
│       | monthly_report           │
├──────────────────────────────────┤
│  ①页面 ②支付 ③通知              │  ← 执行层（6个引擎）
│  ④报告 ⑤库存 ⑥订单              │
├──────────────────────────────────┤
│     FocusingPro 数据后端          │  ← 数据层
└──────────────────────────────────┘
```

### 工作流设计采用引导式对话

工作流支持引导式交互：Tool描述中定义所需信息清单，AI在调用前自动向用户逐步收集必要信息（数据来源、执行方式、范围筛选等），收集完毕后一次性执行。用户体验如同智能表单，但前端是AI对话。

## 六、工作流库（4个通用工作流）

### W1：催款工作流 `payment_collection`

| 项目 | 说明 |
|------|------|
| **触发** | "帮我催一下欠费的" |
| **引导收集** | 数据来源(CSV/手动/已有数据) → 催款范围(全部/按天数筛选) → 催款方式(邮件/邮件+短信/全渠道) → 是否附带支付链接 |
| **编排引擎** | ⑥查询欠款 → ②生成支付链接 → ③发送催款通知 → ④生成催款报告 |
| **分级策略** | 30天内:友善邮件 / 30-60天:邮件+短信 / 60天+:全渠道催款 |
| **输出** | 催款执行报告（发送数/成功数/失败数）+ PDF详细报告URL |
| **行业适配** | 学费催款、房租催款、货款催收 |

### W2：招生工作流 `enrollment`

| 项目 | 说明 |
|------|------|
| **触发** | "帮我启动秋季招生" |
| **引导收集** | 课程信息 → 招生人数 → 学费 → 是否需要面试 → 优惠策略 |
| **编排引擎** | ①生成招生页(带报名表单) → ②支付链接(嵌入报名页) → ③群发招生通知 → ④招生进度报告 |
| **输出** | 招生页URL + 报名数据实时查询 + 进度报告 |
| **行业适配** | 学校招生、培训机构报名、活动报名 |

### W3：发货处理工作流 `order_fulfillment`

| 项目 | 说明 |
|------|------|
| **触发** | "处理今天的订单" |
| **编排引擎** | ⑥查询待处理订单 → ⑤扣减库存 → ③发送发货通知(附物流单号) |
| **输出** | 处理确认（已发货订单数 + 库存变动 + 通知发送状态） |
| **行业适配** | 电商发货、教材发放 |

### W4：月度经营报告工作流 `monthly_report`

| 项目 | 说明 |
|------|------|
| **触发** | "出这个月的经营报告" |
| **编排引擎** | ⑥订单汇总 → ⑤库存汇总 → ④生成PDF报告 |
| **输出** | PDF经营报告URL（收入/支出/库存/订单分析） |
| **行业适配** | 任何有交易数据的业务 |

## 十二、关键决策记录

1. **不做行业垂直Tool，做通用引擎** — 通过参数适配行业，一套引擎覆盖所有
2. **AI调AI优先** — Tool描述英文、标准JSON输入输出、无需认证
3. **无状态优先，有状态可选** — 默认纯执行，`save_to_backend`参数可选持久化
4. **工作流是编排层** — 不是第七个引擎，是调用其他引擎的组合逻辑
5. **FocusingPro对AI不可见** — AI只知道ClawShow，不知道背后的数据存储
6. **Opus做战略，Sonnet写代码** — Claude.ai讨论方向用Opus，VS Code执行用Sonnet
7. **先落地再扩展** — 支付+通知两个引擎先做，配合Florent收款和学校追款两个工作流立即验证，不做没有客户的功能
8. **支付引擎必须含Webhook** — Stripe付款结果自动回写记录，这是和Stripe App竞争的核心差异
9. **通知引擎三渠道** — 邮件+短信+WhatsApp（Twilio统一平台），支持催款分级策略
10. **页面引擎内置GEO** — 生成的每个页面自带llms.txt/Schema/Definition Lead，让AI搜索引擎天然能找到客户
11. **一个MCP Server多处上架** — clawshow.ai/sse是唯一产品，提交到所有分发目录（mcpservers.org/ClawHub/GitHub MCP Registry/LobeHub/PulseMCP/mcp.so）
12. **Stripe Key按namespace管理** — 每个客户的Stripe配置通过namespace隔离，客户密钥不出现在AI对话中
